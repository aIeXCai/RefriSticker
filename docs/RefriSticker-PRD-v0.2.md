# RefriSticker 产品需求文档（PRD）

- 版本：v0.2
- 日期：2026-06-21
- 阶段：MVP / 需求确认稿
- 产品名称：RefriSticker
- 一句话介绍：把旅行照片变成专属冰箱贴插画。

---

## 1. 产品概述

### 1.1 背景

旅行者常会购买当地冰箱贴作为纪念，但现有冰箱贴通常是批量生产的固定图案，无法承载用户亲自拍摄的照片和个人记忆。RefriSticker 允许用户上传旅行照片，选择艺术风格，输入旅行地点，并生成一张可下载、可继续用于实体制作的个性化冰箱贴设计图。

### 1.2 产品目标

MVP 需要验证三个核心假设：

1. 用户愿意把自己的旅行照片转成冰箱贴风格图片。
2. 插画、国风、漫画三种风格具有足够清晰的差异和吸引力。
3. 用户愿意编辑地点文字并下载生成结果，未来可能为高清下载或实体冰箱贴付费。

### 1.3 核心价值

- 个性化：冰箱贴内容来自用户自己的旅行照片。
- 纪念性：通过地点文字记录旅行发生的位置。
- 易用性：无需学习提示词和图片编辑软件。
- 可控性：地点文字由程序排版，不依赖模型生成，避免乱码。

---

## 2. 目标用户

### 2.1 核心用户

- 喜欢旅行、拍照和收集纪念品的年轻用户。
- 喜欢冰箱贴、明信片、手账、文创产品的人群。
- 希望制作情侣、亲子、朋友旅行纪念品的用户。

### 2.2 典型场景

- 旅行结束后，把代表性照片制作成数字冰箱贴。
- 在社交平台分享带地点名称的旅行纪念图。
- 为多次旅行制作统一风格的冰箱贴收藏系列。
- 后续下单制作实体冰箱贴。

---

## 3. MVP 范围

### 3.1 本期包含

1. 首页与产品案例展示。
2. 上传一张旅行照片。
3. 手动裁剪照片。
4. 手动输入旅行地点。
5. 选择插画、国风、漫画三种风格。
6. AI 生成无文字的平面冰箱贴插画底图。
7. 自动添加中文/本地语言地点、英文地点、日期和旅行短句。
8. 分别调整各文字层的内容、大小、位置、颜色和对齐方式。
9. 预览冰箱贴效果。
10. 下载带 RefriSticker 品牌水印的免费图片。
11. 基础错误提示、生成进度和重新生成。

### 3.2 本期暂不包含

- 用户登录和个人作品库。
- 在线支付、点数和会员体系。
- 实体冰箱贴下单及物流。
- 自动识别照片拍摄地点。
- 社区、点赞、评论和公开作品广场。
- 多张照片融合、批量生成和多人协作。
- 用户训练专属 LoRA 或上传自定义风格。

## 5. 页面与功能需求

### 5.1 首页

#### 目标

在 5 秒内让用户理解产品价值，并开始制作。

#### 页面模块

1. 顶部导航
   - RefriSticker Logo。
   - “开始制作”主按钮。
   - “作品示例”“使用方法”锚点。

2. 首屏 Hero
   - 标题：把旅行照片，变成专属冰箱贴。
   - 副标题：上传照片，选择风格，记录地点，生成只属于你的旅行纪念。
   - 主按钮：制作我的冰箱贴。
   - 右侧或下方展示原图与成品的前后对比。

3. 三种风格展示
   - 插画：清新复古旅行海报。
   - 国风：工笔线条、矿物色、纸张肌理。
   - 漫画：清晰线稿、平涂和分明阴影。

4. 三步使用说明
   - 上传旅行照片。
   - 选择喜欢的风格。
   - 添加地点并下载。

5. 页脚
   - 隐私说明。
   - 用户协议占位。
   - 联系方式占位。

### 5.2 创建页：上传照片

#### 功能

- 支持 JPG、JPEG、PNG、WEBP。
- 单张图片最大限制：`{{MAX_UPLOAD_MB}} MB`。
- 推荐图片边长：384～3072 像素。
- 支持点击上传、拖拽上传和移动端相册选择。
- 上传后进入 4:5 裁剪框。
- 支持缩放、移动、旋转和重新上传。

#### 校验

- 文件格式不支持时显示明确提示。
- 文件过大时提示压缩或重新选择。
- 图片过小可能影响效果时允许继续，但显示风险提示。

### 5.3 创建页：地点与风格

#### 地点输入

- 必填项。
- 默认占位：例如“京都，日本”或“杭州，中国”。
- 最大长度：中文 20 字或英文 40 字。
- 支持中文、英文、日文和数字。
- 第一版不调用地图服务，不自动校验地点真实性。

#### 补充文字

- 英文地点：可选；示例 `KYOTO, JAPAN`，最大 40 个字符。
- 日期：可选；支持日期选择器和手动输入，示例 `2026.06.21`、`JUNE 2026`。
- 旅行短句：可选；示例“风吹过京都的夏天”、`OUR FIRST TRIP`，中文最大 30 字、英文最大 60 个字符。
- 用户可以单独隐藏任意补充文字层。

#### 风格选择

| 风格 | Style ID | 用户描述 | 视觉特征 |
|---|---|---|---|
| 插画 | `illustration` | 清新复古的旅行海报插画 | 简化色块、纸张颗粒、明亮低饱和配色 |
| 国风 | `chinese` | 含蓄雅致的东方旅行绘画 | 工笔轮廓、矿物色、宣纸肌理、留白 |
| 漫画 | `comic` | 鲜明有活力的漫画旅行画面 | 清晰线稿、平涂、赛璐璐阴影、适度夸张 |

#### 生成按钮

- 按钮文案：生成冰箱贴。
- 上传成功、地点已填写、风格已选择后才可点击。
- 点击后锁定重复提交，进入生成状态。

### 5.4 生成状态页

- 显示分阶段进度文案：
  - 正在读取照片……
  - 正在绘制旅行插画……
  - 正在制作冰箱贴……
- 支持取消任务（若模型 API 支持）。
- 超过 `{{GENERATION_TIMEOUT_SECONDS}}` 秒后提示任务仍在进行，可继续等待。
- 失败后允许原参数重试，不要求重新上传。

### 5.5 编辑与预览页

#### 图层结构

```text
最上层：地点、英文地点、日期、旅行短句
中间层：白边、圆角、装饰与阴影
最下层：AI 生成的平面插画底图
```

#### 文字编辑

- 分别编辑地点、英文地点、日期和旅行短句。
- 每个文字层均可独立显示、隐藏、选择和移动。
- 拖动改变位置。
- 控制点或滑块调整字号。
- 字号数值输入。
- 颜色选择：深墨、海军蓝、砖红、暖白等预设色。
- 左对齐、居中、右对齐。
- 一键水平居中。
- 快速位置：顶部、画面内底部、留白区居中。
- 撤销和重做，至少支持最近 20 步。
- 文字不能移出安全打印区域；超出时显示警示线。

#### 推荐实现

- 前端编辑器：`{{CANVAS_LIBRARY}}`，候选为 Fabric.js 或 Konva.js。
- 所有位置与字号使用归一化比例保存，避免移动端预览尺寸影响高清导出。

#### 操作按钮

- 下载图片。
- 重新生成。
- 切换风格。
- 更换照片。

### 5.6 下载

- 预览尺寸：`{{PREVIEW_WIDTH}} × {{PREVIEW_HEIGHT}}`。
- 高清尺寸：`{{EXPORT_WIDTH}} × {{EXPORT_HEIGHT}}`。
- 默认 PNG。
- MVP 免费下载图固定带 RefriSticker 品牌水印。
- 水印由网站在导出阶段添加，不交给图片模型生成。
- 水印放置在右下角安全区，保持可辨认但不遮挡地点文字。
- 下载前应在服务端或高清离屏 Canvas 中重新合成，避免低分辨率拉伸。

## 6. 视觉设计方向

### 6.1 品牌气质

- 清新、明亮、轻松、有旅行感。
- 像旅行手账与现代文创商店的结合。
- 避免过度科技感、黑色 AI 工具感和复杂渐变。

### 6.2 建议配色

| 用途 | 颜色 | Hex |
|---|---|---|
| 页面背景 | 云朵白 | `#F8FBF7` |
| 主品牌色 | 湖水薄荷绿 | `#69C7B8` |
| 主按钮深色 | 松石绿 | `#239A8A` |
| 辅助色 | 晴空蓝 | `#8BC9EA` |
| 暖色点缀 | 杏桃橙 | `#FFB77D` |
| 文本主色 | 深青灰 | `#243B3A` |
| 卡片背景 | 暖白 | `#FFFDF8` |

- 大面积留白与柔和浅色背景。
- 卡片使用 20～28px 圆角。
- 阴影轻柔，不使用厚重黑影。

- 按钮文字直接、亲切，例如“开始制作”“换一种风格”。
- 移动端优先，桌面端内容最大宽度建议 1200px。

## 7. AI 生成方案

### 7.1 核心原则

模型只生成无文字的平面插画底图。地点名称、日期、边框、圆角和阴影由网站程序添加，避免模型乱码，并保证所有作品版式一致。

### 7.2 API 占位配置

```text
MODEL_PROVIDER={{MODEL_PROVIDER}}
MODEL_ID={{IMAGE_EDIT_MODEL_ID}}
API_BASE_URL={{IMAGE_API_BASE_URL}}
API_ENDPOINT={{IMAGE_EDIT_ENDPOINT}}
API_KEY_ENV={{IMAGE_API_KEY_ENV}}
MODEL_REGION={{MODEL_REGION}}
MODEL_PRICE_PER_IMAGE={{MODEL_PRICE_PER_IMAGE}}
MODEL_FREE_QUOTA={{MODEL_FREE_QUOTA}}
```

候选模型在开发阶段确认，首版接口层不得绑定某一家供应商。

### 7.3 内部统一请求结构

```json
{
  "sourceImageUrl": "{{SIGNED_SOURCE_IMAGE_URL}}",
  "style": "illustration",
  "location": "京都，日本",
  "promptVersion": "v1",
  "output": {
    "aspectRatio": "4:5",
    "count": 1,
    "watermark": false
  }
}
```

### 7.4 模型适配层

```ts
interface ImageModelAdapter {
  createEditJob(input: GenerateInput): Promise<{ jobId: string }>;
  getJob(jobId: string): Promise<GenerateJob>;
  cancelJob?(jobId: string): Promise<void>;
}
```

以后更换模型时，只替换 Adapter，不改动页面和业务数据结构。

### 7.5 图片生成链路

```text
浏览器压缩预览图
  ↓
后端生成对象存储签名上传地址
  ↓
用户直传原图至对象存储
  ↓
创建生成任务并拼装 Prompt
  ↓
调用 {{MODEL_PROVIDER}} 图片编辑 API
  ↓
轮询或 Webhook 接收结果
  ↓
保存无文字底图
  ↓
前端 Canvas 叠加边框和地点文字
```

## 8. Prompt 设计

### 8.1 Prompt 组装方式

Prompt 不直接由用户输入，也不放在聊天模型的 System Prompt 中。后端根据用户选择的 Style ID，从版本化模板中组装：

```text
最终 Prompt = 基础保真模板 + 风格模板 + 构图模板 + 禁止项
```

Prompt 文件应纳入代码版本管理：

```text
/prompts
  base-v1.txt
  illustration-v1.txt
  chinese-v1.txt
  comic-v1.txt
```

### 8.2 基础 Prompt

```text
请将用户上传的旅行照片转换成一张高完成度的旅行纪念插画。

严格保留原照片中可辨认的拍摄视角、主体、建筑结构、道路关系、自然景物以及人物和车辆的数量与相对位置，使熟悉该地点的人仍然能够辨认原场景。保留原照片的时间、天气和整体情绪，不要添加照片中不存在的人物、建筑、车辆、飞鸟或标志性物体。

整张画面必须使用统一的艺术语言，明确呈现为人工绘制的旅行插画，避免只对照片应用滤镜。

输出4:5竖版平面设计稿。插画占上方约82%，下方约18%保留为完全空白的暖浅色区域，供网站后续添加地点文字。

只输出平面正视图。不要生成实体冰箱贴样机、圆角产品外壳、桌面、背景环境、立体厚度或投影。

不要生成任何中文、英文、字母、数字、Logo、签名、印章、水印、“AI生成”标识或乱码。
```

### 8.3 插画风 Prompt

```text
风格采用清新复古的旅行海报插画：清晰而简化的轮廓和色块，平涂与丝网印刷质感，控制在6至8种主要颜色，配色明亮但不过度饱和，加入细腻的纸张颗粒。树木、天空、人物与建筑使用统一的编辑插画语言，具有现代文创商品和收藏海报的精致感。
```

### 8.4 国风 Prompt

```text
风格采用当代国风工笔旅行插画：细致而克制的墨线轮廓，雅致的矿物色与低饱和青绿、赭石、米白配色，适量宣纸肌理和东方留白。保留真实地点和现代物体，不要把现代旅行场景改造成古代建筑或历史人物场景。整体像现代博物馆文创，而不是传统山水画复制品。
```

### 8.5 漫画风 Prompt

```text
风格采用明快精致的旅行漫画插画：干净有力的线稿，清晰的平涂色块，赛璐璐阴影，适度夸张但保持地点、人物身份与空间结构可辨认。配色活泼清爽，避免密集网点、对话框、拟声词和分镜边框，整体像一张单幅收藏级旅行漫画海报。
```

### 8.6 禁止项

```text
禁止：文字、字母、数字、Logo、水印、签名、印章、乱码、重复人物、额外肢体、畸形人脸、凭空增加物体、改变地标结构、产品样机、桌面背景、3D厚度、强烈摄影滤镜。
```

### 8.7 Prompt 调试方法

- 准备 10 张固定测试图片：城市街景3张、自然风景2张、建筑2张、人物旅行照2张、夜景1张。
- 每种风格至少生成两轮。
- 固定测试维度：地点保真、人物保真、风格强度、无乱码、文字留白、画面完成度。
- 每次只修改一个变量，并记录 Prompt 版本和模型版本。
- MVP 发布前，每种风格至少达到 80% 可接受生成率。

## 12. 验收标准

1. 用户能在移动端上传并裁剪一张照片。
2. 用户必须输入地点并选择三种风格之一。
3. 服务端能够使用占位模型适配层创建和查询任务。
4. 成功后展示无模型文字的平面插画底图。
5. 地点、英文地点、日期和旅行短句均由 Canvas 添加，可独立拖动、隐藏和调整字号。
6. 刷新前的当前编辑状态自动保存在浏览器中。
7. 下载图片中的文字、边框和底图位置与预览一致。
8. 模型失败、超时和违规拒绝均有可理解提示。
9. API Key 不出现在前端代码、网络响应或日志中。
10. 原图和生成结果能按保留策略自动删除。

## 14. 已确认的产品决策

1. MVP 不提供登录，不保留历史作品。
2. 免费下载图片带 RefriSticker 品牌水印。
3. 文字层支持地点、英文地点、日期和旅行短句。
4. 近期只跑通数字图片生成、编辑和下载闭环，不接入实体冰箱贴制作。

### 仍待后续确认

1. 首发用户主要来自中国大陆，还是也需要面向海外用户？这会影响模型、存储和域名部署。
2. 用户照片中有人物时，产品是否强调“保留人脸”，还是更偏向整体风格化？

## 15. 当前默认决策

在上述问题确认前，MVP 默认采用：

- 中国大陆用户优先。
- 移动端优先。
- 无需登录，关闭页面后不保留云端历史作品。
- 免费下载图片带 RefriSticker 品牌水印。
- 地点主标题必填；英文地点、日期和旅行短句可选。
- 使用第三方图片编辑 API，不自建 GPU。
- 图片默认保存 24 小时后删除。
- 先完成数字图片生成、文字编辑和下载闭环，不接支付和实体制作。
